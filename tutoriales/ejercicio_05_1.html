🧠 ¿Qué son las Funciones?

Concepto de Función

Una función es como una máquina especializada que:

  • 🔵 Recibe ingredientes (parámetros)
  • 🔧 Procesa esos ingredientes (código)
  • 📦 Devuelve un producto terminado (resultado)

🔧 Anatomía de una Función

Estructura Básica de una Función

def nombre_funcion(parametro1, parametro2): """Documentación de la función""" # Código de la función resultado = parametro1 + parametro2 return resultado
def Palabra clave que indica que vamos a definir una función
nombre_funcion El nombre que le damos a nuestra función (debe ser descriptivo)
parámetros Los datos que la función necesita para trabajar (van entre paréntesis)
: Los dos puntos indican que empieza el cuerpo de la función
return Devuelve el resultado de la función (opcional)

🔍 Conceptos Avanzados de Funciones

Parámetros vs Argumentos

  • Parámetros: Las variables en la definición de la función
  • Argumentos: Los valores reales que pasamos al llamar la función
def saludar(nombre, edad): # nombre y edad son PARÁMETROS return f"Hola {nombre}, tienes {edad} años" saludar("Juan", 25) # "Juan" y 25 son ARGUMENTOS

Retorno de Valores

Las funciones pueden devolver diferentes tipos de datos:

# Función que no retorna nada (None implícito) def imprimir_mensaje(msg): print(msg) # No hay return, devuelve None # Función que retorna un valor def calcular_area(base, altura): return base * altura / 2 # Función que retorna múltiples valores def obtener_estadisticas(numeros): return min(numeros), max(numeros), sum(numeros) minimo, maximo, total = obtener_estadisticas([1, 5, 3, 9, 2])

Alcance de Variables (Scope)

Las variables dentro de una función son locales a esa función:

mensaje_global = "Soy global" def mi_funcion(): mensaje_local = "Soy local" print(mensaje_local) # ✓ Funciona print(mensaje_global) # ✓ Funciona mi_funcion() print(mensaje_global) # ✓ Funciona # print(mensaje_local) # ❌ Error: no existe fuera de la función

🔍 Conceptos Clave

Definición de Funciones

Las funciones son bloques de código reutilizables que realizan una tarea específica.

def nombre_de_la_funcion(parametro1, parametro2): # Código de la función return resultado

Parámetros y Argumentos

Los parámetros son variables que se definen en la función, los argumentos son los valores que se pasan a la función al llamarla.

def saludar(nombre): """Saluda a la persona""" print(f"¡Hola, {nombre}!") saludar("Carlos") # Llamada a la función con el argumento "Carlos"

Retorno de Valores

Las funciones pueden devolver valores usando la palabra clave return.

def sumar(a, b): """Suma dos números y devuelve el resultado""" return a + b resultado = sumar(5, 3) print(f"El resultado es: {{resultado}}") # Imprime: El resultado es: 8

Ámbito de las Variables

Las variables definidas dentro de una función son locales y no se pueden acceder fuera de ella.

def funcion_local(): mensaje_local = "Hola desde la función" print(mensaje_local) # Funciona correctamente funcion_local() try: print(mensaje_local) # Esto fallará, ya que mensaje_local es local except NameError: print("mensaje _local no está definido fuera de la función")

Funciones Anidadas

Las funciones pueden llamarse dentro de otras funciones, lo que permite crear código modular y reutilizable.

def funcion_externa(x): """Función externa que llama a una función interna""" def funcion_interna(y): """Función interna que multiplica por 2""" return y * 2 resultado = funcion_interna(x) print(f"El resultado es: {{resultado}}") funcion_externa(5) # Imprime: El resultado es: 10

Funciones Lambda

Las funciones lambda son funciones anónimas y pequeñas que se pueden definir en una sola línea.

lambda x: x * 2 # Función lambda que multiplica por 2 doblar = lambda x: x * 2 print(f"El doble de 4 es: {{doblar(4)}}") # Imprime: El doble de 4 es: 8

📚 Buenas Prácticas en Funciones

Principios de Diseño de Funciones

Al crear funciones, sigue estas buenas prácticas:

Principio Descripción Ejemplo
Un solo propósito Cada función debe hacer una sola cosa y hacerlo bien ✅ calcular_promedio(lista)
❌ calcular_y_mostrar_promedio(lista)
Nombres descriptivos Usa nombres claros que indiquen lo que hace la función ✅ obtener_datos_usuario()
❌ funcion1()
Parámetros claros Usa parámetros con nombres descriptivos y evita demasiados ✅ enviar_email(destinatario, asunto, mensaje)
❌ enviar_email(a, b, c)
Documentación Incluye docstrings para explicar qué hace la función ✅ def sumar(a, b):
    """Suma dos números"""
❌ def sumar(a, b): pass
Evitar efectos secundarios Las funciones deben ser puras, sin modificar el estado global ✅ def duplicar(x): return x * 2
❌ def duplicar(x): global y; y = x * 2
Pruebas unitarias Escribe pruebas para verificar que las funciones funcionan correctamente ✅ assert sumar(2, 3) == 5
❌ # No hay pruebas

Errores Comunes al Usar Funciones

Evita estos errores comunes:

  • ❌ No documentar las funciones adecuadamente
  • ❌ Usar nombres de funciones poco descriptivos
  • ❌ Crear funciones demasiado largas o complejas
  • ❌ No manejar excepciones dentro de las funciones
  • ❌ No reutilizar código, duplicando lógica en varias funciones

Beneficios de Usar Funciones

Las funciones ofrecen múltiples ventajas en la programación:

  • 🔄 Permiten reutilizar código sin duplicación
  • 🧩 Facilitan la organización y modularización del código
  • 🔍 Mejoran la legibilidad y mantenibilidad del código
  • 🧪 Permiten realizar pruebas unitarias para verificar su funcionamiento
  • 🚀 Facilitan la colaboración en proyectos grandes, dividiendo tareas
  • 📈 Mejoran la escalabilidad del código, permitiendo agregar nuevas funcionalidades fácilmente
