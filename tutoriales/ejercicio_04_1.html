🧠 Conceptos Teóricos: Estructuras Condicionales

¿Qué son las Estructuras Condicionales?

Las estructuras condicionales permiten que nuestro programa tome decisiones. Es como darle a la computadora la capacidad de pensar: "SI pasa esto, ENTONCES haz aquello, SI NO, haz otra cosa".

INICIO
↓
¿Condición?
SÍ →
Acción 1
NO →
Acción 2

Sintaxis de if/elif/else en Python

if condición1: # Código si condición1 es True elif condición2: # Código si condición1 es False y condición2 es True elif condición3: # Código si las anteriores son False y condición3 es True else: # Código si todas las condiciones son False
Estructura Cuándo usar Ejemplo de uso
if Una sola condición Si llueve, lleva paraguas
if...else Dos opciones mutuamente excluyentes Si es mayor de edad, puede votar; si no, debe esperar
if...elif...else Múltiples opciones Calificación: A, B, C, D o F según la nota

🔍 Operadores de Comparación y Lógicos

Operadores de Comparación

Operador Significado Ejemplo Resultado
== Igual a 5 == 5 True
!= Diferente de 5 != 3 True
> Mayor que 5 > 3 True
< Menor que 5 < 3 False
>= Mayor o igual 5 >= 5 True
<= Menor o igual 5 <= 6 True

Operadores Lógicos

  • and: Ambas condiciones deben ser True
  • or: Al menos una condición debe ser True
  • not: Invierte el valor (True → False, False → True)
# Ejemplos de operadores lógicos edad = 25 tiene_licencia = True # AND: ambas deben ser verdaderas if edad >= 18 and tiene_licencia: print("Puedes conducir") # OR: al menos una debe ser verdadera if edad < 18 or not tiene_licencia: print("No puedes conducir")

🔬 Análisis del Flujo de Decisiones

Orden de las Condiciones: El orden importa. Python evalúa las condiciones de arriba hacia abajo y ejecuta el primer bloque que sea True. Las condiciones más específicas deben ir primero.

Buenas Prácticas en Estructuras Condicionales

  • ✅ Condiciones específicas primero: Las más restrictivas van antes que las generales
  • ✅ Evitar redundancia: No repetir condiciones ya evaluadas
  • ✅ Usar elif en lugar de múltiples if: Más eficiente y claro
  • ✅ Incluir un else final: Para manejar casos no previstos
  • ✅ Comentar la lógica compleja: Facilita el mantenimiento
Técnica de Depuración: Agrega print() temporales dentro de cada bloque para ver qué camino está tomando tu código:
if condicion: print("DEBUG: Entró al primer if") # resto del código
